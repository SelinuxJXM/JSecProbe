## 1. 软件安装

### 1.1 系统要求

| 项目 | 最低配置 | 推荐配置 |
|------|---------|---------|
| 操作系统 | Windows 10 64位 | Windows 10/11 64位 |
| 处理器 | Intel Core i3 或同级 | Intel Core i5 或更高 |
| 内存 | 4 GB | 8 GB 或更高 |
| 磁盘空间 | 2 GB 可用空间 | 10 GB 可用空间 |
| 网络 | 可选 | 宽带网络（AI 功能需要） |

### 1.2 安装步骤

#### 方式一：安装包安装（推荐）

1. 双击运行 `等级保护现场测评系统 Setup 1.2.0.exe`
2. 选择安装语言（默认中文）
3. 阅读并接受许可协议
4. 选择安装路径（默认 `C:\Program Files\等级保护现场测评系统`）
5. 选择是否创建桌面快捷方式和开始菜单快捷方式
6. 点击"安装"，等待安装完成
7. 点击"完成"启动软件

#### 方式二：便携版运行

1. 解压 `等级保护现场测评系统 1.2.0.exe` 到任意目录
2. 双击运行即可使用
3. 数据存储在程序同级目录下的 `data` 文件夹中

### 1.3 卸载

1. 打开 Windows"设置" → "应用" → "应用和功能"
2. 搜索"等级保护现场测评系统"
3. 点击"卸载"，按提示完成卸载

**注意**: 卸载不会删除您的项目数据。如需删除数据，请手动删除 `%APPDATA%\mlps-assessment-tool` 目录。

## 4. 项目管理

### 4.1 创建项目

1. 点击左侧导航"项目首页" → "项目列表"
2. 点击右上角"新建项目"按钮
3. 填写项目信息：

| 字段 | 说明 | 是否必填 |
|------|------|---------|
| 项目名称 | 项目的名称 | ✅ |
| 系统名称 | 被测信息系统的名称 | ✅ |
| 保护等级 | 选择安全保护等级（二级/三级/四级） | ✅ |
| 标准体系 | 选择使用的标准（默认 GB/T 22239-2019） | ✅ |
| 扩展要求 | 选择适用的扩展要求（可多选） | ❌ |
| 被测单位 | 被测系统的所属单位 | ❌ |
| 测评工程师 | 项目负责人或主测工程师 | ❌ |
| 测评时间 | 计划测评开始和结束日期 | ❌ |
| 项目描述 | 项目补充说明 | ❌ |

4. 点击"创建"完成

**注意**: 创建项目后，系统会根据所选等级和扩展要求自动加载适用的测评项。

### 4.2 编辑项目

1. 在项目列表中找到目标项目
2. 点击操作列的"编辑"按钮
3. 修改项目信息
4. 点击"保存"

### 4.3 删除项目

1. 在项目列表中找到目标项目
2. 点击操作列的"删除"按钮
3. 确认删除操作

**警告**: 删除项目将同时删除该项目的所有资产、测评记录、问题和证据文件，且不可恢复。请谨慎操作。

### 4.4 项目状态管理

项目支持以下状态：

| 状态 | 说明 | 操作 |
|------|------|------|
| 草稿 | 项目已创建，尚未开始测评 | 点击"开始测评"进入进行中 |
| 进行中 | 测评工作正在进行 | 点击"完成"标记为已完成 |
| 已完成 | 测评工作全部完成 | 点击"归档"归档项目 |
| 已归档 | 项目已归档，只读 | 点击"恢复"恢复为已完成 |

## 5. 系统构成（资产管理）

### 5.1 功能概述

系统构成模块用于管理被测系统的资产信息，是现场核查的基础。

### 5.2 资产类别

系统支持以下 10 类资产：

| 类别代码 | 类别名称 | 说明 |
|---------|---------|------|
| server_storage | 服务器/存储 | 物理服务器、存储设备 |
| dbms | 数据库管理系统 | Oracle、MySQL、PostgreSQL 等 |
| network_device | 网络设备 | 交换机、路由器等 |
| security_device | 安全设备 | 防火墙、WAF、IPS 等 |
| business_app | 业务应用系统 | 业务系统、Web 应用 |
| terminal | 终端设备 | PC、笔记本 |
| management_platform | 管理平台 | 运维管理、安全管理平台 |
| machine_room | 机房设施 | 机房、UPS、空调 |
| data_resource | 数据资源 | 数据库、文件数据 |
| network_boundary | 网络边界 | 网络边界防护设备 |

### 5.3 添加资产

1. 进入项目后，点击左侧导航"系统构成"
2. 点击右上角"添加资产"按钮
3. 选择资产类别
4. 填写资产信息：

| 字段 | 说明 | 是否必填 |
|------|------|---------|
| 资产名称 | 资产的名称 | ✅ |
| IP 地址 | 资产的 IP 地址 | ❌ |
| 操作系统 | 运行的操作系统 | ❌ |
| 版本号 | 软件版本号 | ❌ |
| 重要程度 | 高/中/低 | ✅ |
| 是否测评对象 | 是/否 | ✅ |
| 数量 | 资产数量 | ✅ |
| 责任人 | 资产负责人 | ❌ |

5. 点击"保存"

### 5.4 批量导入资产

1. 在系统构成页面，点击"导入 Excel"
2. 选择 Excel 文件
3. 系统自动解析并导入资产

**Excel 格式要求**:
- 第一行为表头，包含：名称、类别、IP、操作系统、版本、重要程度、数量、责任人等列
- 类别需使用系统定义的类别代码

### 5.5 编辑/删除资产

- 点击资产行操作列的"编辑"按钮修改资产信息
- 点击"删除"按钮删除单个资产
- 勾选多个资产后点击"批量删除"可批量删除

## 6. 现场核查

### 6.1 功能概述

现场核查是系统的核心功能，用于执行测评项的核查工作，记录测评结果和证据。

### 6.2 进入现场核查

1. 在项目列表中选择目标项目
2. 点击"进入项目"或点击项目名称
3. 在项目内导航中点击"现场核查"

### 6.3 安全域选择

现场核查页面左侧显示安全域列表：

| 安全域 | 说明 |
|--------|------|
| 安全物理环境 | 机房、物理访问、防盗窃等 |
| 安全通信网络 | 网络架构、通信传输等 |
| 安全区域边界 | 边界防护、访问控制等 |
| 安全计算环境 | 主机安全、应用安全、数据安全 |
| 安全管理中心 | 系统管理、审计管理等 |
| 安全管理制度 | 安全策略、管理制度等 |
| 安全管理机构 | 人员配备、岗位职责等 |
| 安全管理人员 | 人员录用、培训等 |
| 安全建设管理 | 项目管理、工程实施等 |
| 安全运维管理 | 运维管理、变更管理等 |

点击安全域可查看该域下的测评项列表。

### 6.4 测评项列表

右侧显示当前安全域下的测评项列表，包含以下信息：

- **序号**: 测评项编号
- **控制点**: 测评项所属控制点
- **要求**: 测评项的具体要求内容
- **符合情况**: 当前测评结果（符合/部分符合/不符合/不适用/未测试）
- **操作**: 查看记录、编辑

### 6.5 执行核查

#### 6.5.1 层面测评

1. 在测评项列表中，点击目标测评项的"核查"按钮
2. 在弹出的核查窗口中：

**基本信息区**:
- 显示控制点、要求内容

**核查结果区**:
- **核查方法**: 选择"访谈"/"检查"/"测试"
- **符合情况**: 选择"符合"/"部分符合"/"不符合"/"不适用"
- **命令输出**: 输入核查命令及其输出结果
- **证据描述**: 输入证据的文字描述

**证据上传区**:
- 点击"上传截图"选择图片文件
- 点击"上传文件"选择其他类型文件
- 已上传文件显示在列表中，可预览或删除

3. 点击"保存"完成核查记录

#### 6.5.2 资产测评

1. 在页面顶部切换到"资产测评"标签
2. 选择目标资产
3. 系统根据资产类别自动匹配适用测评项
4. 按照层面测评的流程执行核查
5. 记录自动关联到所选资产

### 6.6 查看核查历史

1. 点击测评项的"记录"按钮
2. 显示该测评项的所有历史核查记录
3. 可查看每次核查的详细信息

### 6.7 进度查看

页面顶部显示当前项目的测评进度：

- **总测评项数**: 当前项目适用的测评项总数
- **已测评数**: 已填写结果的测评项数
- **符合数**: 判定为"符合"的测评项数
- **符合率**: 符合数 / 已测评数 × 100%
- **进度条**: 可视化展示完成进度

### 6.8 导出核查记录

1. 点击页面右上角"导出 Excel"
2. 选择导出范围（全部/当前安全域/按资产）
3. 选择保存路径
4. 系统生成包含核查记录的 Excel 文件

**导出内容**:
- 按安全域分 Sheet
- 包含序号、控制点、要求、结果记录、符合情况、证据文件
- 截图自动嵌入到 Excel 中

### 6.9 导入核查记录

1. 点击页面右上角"导入 Excel"
2. 选择之前导出的 Excel 文件
3. 系统自动解析并更新核查记录

## 7. AI 智能辅助

### 7.1 功能概述

AI 智能辅助功能利用大语言模型，帮助分析测评证据并生成测评结论。

### 7.2 配置 AI

首次使用 AI 功能前，需要先进行配置：

1. 进入"系统设置" → "AI 配置"
2. 填写配置信息：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| API 地址 | AI 服务的 API 地址 | `https://api.openai.com` |
| API Key | API 密钥 | `sk-xxxxxxxxxxxxxxxx` |
| 模型名称 | 使用的模型 | `gpt-4o-mini` |
| 温度参数 | 控制输出随机性（0-1） | `0.3` |

3. 点击"测试连接"验证配置是否正确
4. 点击"保存"保存配置

**支持的 API**:
- OpenAI API
- 任何兼容 OpenAI API 格式的服务（如 Azure OpenAI、本地部署的模型等）

### 7.3 智能分析

1. 在现场核查页面，点击测评项的"AI 分析"按钮
2. 在弹出的分析窗口中：
   - 系统自动填充控制点和要求内容
   - 输入核查结果（命令输出、检查发现等）
   - 可选：上传相关截图
3. 点击"开始分析"
4. AI 返回分析结果：
   - **关键证据点**: 从证据中提取的关键信息
   - **符合情况**: AI 判定的符合性
   - **测评结论**: 完整的结论文字
5. 点击"应用到记录"将 AI 分析结果保存到核查记录

### 7.4 批量分析

1. 在现场核查页面，点击右上角"批量 AI 分析"
2. 在弹出的窗口中：
   - 选择要分析的测评项（可多选）
   - 上传相关截图（可多张）
3. 点击"开始分析"
4. 查看分析进度：
   - 编码阶段：压缩和编码图片（15-50%）
   - 发送阶段：提交给 AI（60%）
   - 接收阶段：等待 AI 响应（75%）
   - 解析阶段：解析 AI 返回结果（90%）
   - 完成：分析完成（100%）
5. 查看分析结果：
   - 系统自动将截图与测评项匹配
   - 显示每个测评项的分析结论
6. 点击"应用全部"或逐条应用结果

### 7.5 AI 对话

1. 点击左侧导航"AI 智能辅助"
2. 在对话框中输入问题
3. AI 根据上下文回答问题
4. 支持多轮对话

**使用场景**:
- 询问某项要求的具体含义
- 询问某类系统的配置方法
- 询问整改建议

## 8. 问题管理

### 8.1 功能概述

问题管理模块用于跟踪和管理测评发现的不符合项。

### 8.2 生成问题

#### 自动生成

1. 在现场核查中，将测评项判定为"不符合"
2. 系统自动生成对应的问题记录
3. 问题标题格式：`[控制点] 不符合要求`

#### 手动创建

1. 进入项目后，点击左侧导航"问题清单"
2. 点击右上角"新建问题"
3. 填写问题信息：

| 字段 | 说明 | 是否必填 |
|------|------|---------|
| 安全域 | 所属安全域 | ✅ |
| 控制点 | 关联控制点 | ✅ |
| 问题标题 | 问题简述 | ✅ |
| 问题描述 | 详细描述 | ✅ |
| 风险等级 | 高/中/低 | ✅ |
| 整改建议 | 建议的整改措施 | ❌ |
| 整改期限 | 要求整改完成日期 | ❌ |
| 责任人 | 整改责任人 | ❌ |

4. 点击"保存"

### 8.3 问题列表

问题列表显示所有问题，支持以下操作：

- **筛选**: 按风险等级、状态、安全域筛选
- **排序**: 按创建时间、风险等级排序
- **搜索**: 按标题或描述搜索

### 8.4 问题详情

点击问题行进入详情页面，可查看和编辑：

- 问题基本信息
- 关联的测评记录和证据
- 整改记录
- 证据文件

### 8.5 问题状态流转

```
待处理 → 整改中 → 已整改 → 已关闭
  │                    │
  └────────────────────┘
      (可回退)
```

- **待处理**: 问题已创建，尚未开始整改
- **整改中**: 正在执行整改措施
- **已整改**: 整改措施已完成，待验证
- **已关闭**: 验证通过，问题关闭

### 8.6 批量操作

1. 勾选多个问题
2. 点击"批量更新状态"批量修改状态
3. 点击"批量删除"批量删除问题

### 8.7 导出问题清单

1. 点击右上角"导出 Excel"
2. 选择保存路径
3. 系统生成包含所有问题信息的 Excel 文件

## 9. 报告生成

### 9.1 功能概述

报告生成功能根据项目数据自动生成 PDF 格式的测评报告。

### 9.2 生成报告

1. 进入目标项目
2. 点击左侧导航"报告生成"
3. 点击"生成报告"按钮
4. 系统自动收集项目数据并生成 PDF
5. 生成完成后显示报告路径

### 9.3 报告内容

生成的报告包含以下内容：

1. **封面**: 项目名称、系统名称、保护等级、测评日期
2. **项目信息**: 被测单位、测评工程师、测评时间
3. **测评结果统计**:
   - 测评项总数
   - 符合项数量
   - 不符合项数量
   - 符合率
4. **问题清单**: 不符合项的详细信息表格

### 9.4 查看历史报告

- 报告生成后自动保存到项目目录
- 在报告列表中可以查看和打开历史报告

## 12. 常见问题

### 12.1 安装与启动

**Q: 安装时提示"无法写入文件"怎么办？**
A: 请关闭正在运行的软件实例，或重启电脑后重新安装。如果问题仍然存在，请以管理员身份运行安装程序。

**Q: 软件启动后白屏怎么办？**
A: 请尝试以下操作：
1. 关闭软件
2. 删除 `%APPDATA%\mlps-assessment-tool` 目录
3. 重新启动软件

**Q: 如何查看软件版本？**
A: 点击"系统设置" → "关于"查看软件版本信息。

### 12.2 登录与账号

**Q: 忘记密码怎么办？**
A: 请联系系统管理员重置密码。管理员可在"系统设置" → "用户管理"中重置用户密码。

**Q: 首次登录提示修改密码？**
A: 出于安全考虑，首次登录必须修改密码。请设置一个安全的密码（建议包含大小写字母、数字和特殊字符）。

### 12.3 数据管理

**Q: 数据保存在哪里？**
A: 数据默认保存在 `%APPDATA%\mlps-assessment-tool` 目录。可在"系统设置" → "存储设置"中查看和修改。

**Q: 如何备份数据？**
A: 系统支持自动备份和手动备份。建议定期手动备份重要数据。备份文件为 SQLite 数据库文件。

**Q: 如何迁移数据到另一台电脑？**
A: 方法一：使用"数据备份"功能创建备份文件，在另一台电脑上恢复。方法二：直接复制 `%APPDATA%\mlps-assessment-tool` 目录到另一台电脑。

### 12.4 AI 功能

**Q: AI 功能无法使用怎么办？**
A: 请检查：
1. 是否已配置 AI（系统设置 → AI 配置）
2. API 地址和 API Key 是否正确
3. 网络连接是否正常
4. 点击"测试连接"验证配置

**Q: AI 分析结果不准确怎么办？**
A: AI 分析结果仅供参考，请结合专业判断。您可以：
1. 提供更详细的证据信息
2. 上传相关截图辅助分析
3. 手动修改分析结果

**Q: 支持哪些 AI 模型？**
A: 系统兼容 OpenAI API 格式，支持：
- OpenAI GPT 系列
- Azure OpenAI
- 本地部署的兼容模型（如 Ollama、vLLM 等）

### 12.5 其他

**Q: 如何更新软件？**
A: 系统支持自动更新。当有新版本时，会弹出更新提示。也可在"系统设置" → "关于"中手动检查更新。

**Q: 软件卡顿怎么办？**
A: 请尝试：
1. 关闭不需要的窗口
2. 减少同时打开的项目数量
3. 清理历史截图和临时文件
4. 重启软件

**Q: 如何反馈问题？**
A: 请通过以下方式反馈：
1. 软件内"帮助" → "反馈问题"
2. 联系技术支持邮箱
