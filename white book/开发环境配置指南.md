## 1. 环境准备

### 1.1 Node.js

本项目使用 Node.js v24.18.0 开发，建议使用相同或更高版本。

**安装步骤**:

1. 访问 [Node.js 官网](https://nodejs.org/) 下载 LTS 版本（推荐 v20.x 或更高）
2. 运行安装程序，按向导完成安装
3. 验证安装：

```powershell
node -v
npm -v
```

### 1.2 Git

下载安装 [Git for Windows](https://git-scm.com/download/win)，安装时选择"Use Git from the Windows Command Prompt"以便在 PowerShell 中使用。

验证安装：

```powershell
git --version
```

### 1.3 代码编辑器

推荐使用 **Visual Studio Code**，安装以下扩展提升开发效率：

| 扩展 | 用途 |
|------|------|
| Vue.volar | Vue 3 语言支持（TypeScript） |
| dbaeumer.vscode-eslint | ESLint 集成 |
| esbenp.prettier-vscode | Prettier 格式化 |
| bradlc.vscode-tailwindcss | CSS 智能提示 |
| ms-vscode.vscode-typescript-next | TypeScript 增强 |

### 1.4 Python（可选）

`better-sqlite3` 是原生 NAPI 模块。如果没有预编译二进制文件，需要 Python 和 Visual Studio Build Tools 从源码编译。如果 `npm install --ignore-scripts` + `electron-rebuild` 可以工作（大多数情况下如此），则不一定需要 Python。

**如需安装 Python**:

1. 安装 Python 3.10+，勾选 "Add Python to PATH"
2. 安装 Visual Studio Build Tools（勾选"使用 C++ 的桌面开发"）

## 3. 安装项目依赖

### 3.1 配置 npm 镜像（推荐）

项目根目录已有 `.npmrc` 文件，配置了 electron 镜像。检查内容：

```
ELECTRON_MIRROR=https://npmmirror.com/mirrors/electron/
ELECTRON_BUILDER_BINARIES_MIRROR=https://npmmirror.com/mirrors/electron-builder-binaries/
```

### 3.2 安装依赖

**方式一：直接安装（可能需要 Python）**

```powershell
npm install
```

**方式二：跳过编译脚本（推荐，更易成功）**

```powershell
npm install --ignore-scripts
```

此方式跳过了 `prebuild-install` 和 `node-gyp` 的自动编译，稍后通过 `electron-rebuild` 为 Electron 版本重新编译原生模块。

## 4. 编译原生模块

项目使用 `better-sqlite3` 作为数据库引擎，它是原生 NAPI 模块，需要针对 Electron 使用的 Node.js 版本重新编译。

### 4.1 使用 electron-rebuild

执行以下命令为 Electron 编译原生模块：

```powershell
npx electron-rebuild
```

如果 PowerShell 执行策略阻止 npm 脚本运行，先设置策略：

```powershell
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned -Force
npx electron-rebuild
```

成功后会看到：

```
✔ Rebuild Complete
```

### 4.2 验证编译结果

检查 `node_modules/better-sqlite3/build/Release/` 目录下是否存在 `better_sqlite3.node` 文件。

## 5. 数据库初始化

应用首次启动时会自动完成数据库创建和迁移：

1. 应用启动后，`initDatabase()` 函数会执行 `electron/db/migrate.ts`
2. 迁移文件位于 `electron/db/migrations/` 目录下（`001_init.sql`, `002_add_missing_fields.sql`）
3. 种子数据自动加载（`electron/db/seeds/`），内置 GB/T 22239-2019 标准测评项和常用核查命令

**数据库文件位置**: `%APPDATA%\mlps-assessment-tool\database\main.db`

### 5.1 手动执行迁移（如需）

```powershell
npm run db:migrate
```

### 5.2 重新生成迁移

如果修改了数据库 schema（`electron/db/schema.ts`），需要重新生成迁移：

```powershell
npm run db:generate
```

## 6. 启动开发服务器

### 6.1 开发模式

```powershell
npm run dev
```

这将同时启动：

- **Vite 开发服务器**（端口 5173）：前端 Vue 应用，支持 HMR
- **Electron 主进程**：连接开发服务器，可使用 DevTools 调试

### 6.2 开发工具

- **Chrome DevTools**: Electron 窗口中自动打开，用于调试渲染进程进程（Console、Network、Elements 等）
- **Vite HMR**: 修改 Vue 组件后浏览器自动热更新
- **主进程调试**: 在 VS Code 中配置 `launch.json` 可断点调试 Electron 主进程

### 6.3 VS Code 调试配置（可选）

在 `.vscode/launch.json` 中添加：

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Electron: Main",
      "type": "node",
      "request": "launch",
      "cwd": "${workspaceFolder}",
      "runtimeExecutable": "${workspaceFolder}/node_modules/.bin/electron",
      "windows": {
        "runtimeExecutable": "${workspaceFolder}/node_modules/.bin/electron.cmd"
      },
      "args": ["."],
      "outputTrace": true,
      "console": "integratedTerminal"
    }
  ]
}
```

## 8. 开发工作流

### 8.1 NPM 脚本命令

| 命令 | 说明 |
|------|------|
| `npm run dev` | 启动开发服务器（前端 + Electron） |
| `npm run build` | 完整构建（类型检查 + Vite 构建 + Electron 打包） |
| `npm run build:win` | Windows 平台打包 |
| `npm run build:portable` | Windows 便携版打包 |
| `npm run rebuild` | 重新编译原生模块 |
| `npm run db:generate` | 重新生成数据库迁移文件 |
| `npm run db:migrate` | 手动执行数据库迁移 |

### 8.2 修改代码后的热更新

| 修改类型 | 自动更新方式 |
|---------|-------------|
| Vue 组件（`src/**/*.vue`） | Vite HMR 实时刷新样式和模板 |
| TypeScript/Javascript（`src/`） | Vite HMR 重新编译 |
| Electron 主进程（`electron/main/`, `electron/ipc/`） | `vite-plugin-electron` 重启 Electron |
| Preload 脚本（`electron/preload/`） | `vite-plugin-electron` 重新加载窗口 |
| Vite 配置（`vite.config.ts`） | 需重启 `npm run dev` |
| package.json | 需重启 `npm run dev` |

### 8.3 IPC 通信开发模式

新增 IPC 通道的步骤：

**1. 在 `shared/types.ts` 的 `ApiBridge` 接口中定义类型**

```typescript
ai: {
  // ...已有方法
  newMethod: (params: SomeParams) => Promise<IpcResponse<SomeResult>>;
};
```

**2. 在 `electron/ipc/` 新增或修改 IPC 处理器**

```typescript
// electron/ipc/ai.ipc.ts
ipcMain.handle('ai:newMethod', (event, params) =>
  wrap(async () => {
    // 业务逻辑
    return result;
  })
);
```

**3. 在 `electron/preload/index.ts` 暴露 API**

```typescript
// 在 api 对象中
ai: {
  // ...已有方法
  newMethod: (params) => ipcRenderer.invoke('ai:newMethod', params),
},
```

**4. 在 Vue 组件中调用**

```typescript
const res = await window.api.ai.newMethod(params);
if (res.success) {
  // 处理结果
}
```

### 8.4 样式开发

项目使用 SCSS + CSS Variables，全局变量定义在 `src/styles/variables.scss`，已通过 Vite 配置自动注入：

```typescript
// vite.config.ts
css: {
  preprocessorOptions: {
    scss: {
      additionalData: `@use "@/styles/variables.scss" as *;`,
    },
  },
},
```

组件内直接使用，无需手动 import：

```vue
<style scoped lang="scss">
.my-component {
  color: $primary-color;
  padding: $spacing-base;
}
</style>
```

## 9. 生产构建

### 9.1 完整构建（推荐）

```powershell
npm run build
```

包含步骤：
1. `vue-tsc --noEmit` — TypeScript 类型检查
2. `vite build` — 构建前端代码到 `dist-renderer/`
3. `electron-builder` — 打包为 Windows 安装包和便携版

### 9.2 Windows 平台构建

```powershell
npm run build:win
```

产物输出到 `dist/` 目录：

- `JSecProbe Setup v1.2.4.exe` — NSIS 安装包
- `JSecProbe v1.2.4.exe` — 便携版（Portable）

### 9.3 仅构建前端

如需仅构建前端代码（用于部署为 Web 应用），修改 `vite.config.ts` 移除 electron 插件后执行 `vite build`，产物在 `dist-renderer/`。注意：IPC 调用在纯 Web 环境下不可用。

## 10. 常见问题排查

### 10.1 npm install 失败

**问题**: `better-sqlite3` 编译失败

```
npm error path .../better-sqlite3
npm error gyp ERR! find Python
```

**解决**:
```powershell
npm install --ignore-scripts
npx electron-rebuild
```

如仍失败，安装 Python 和 VS Build Tools 后重试 `npm install`。

### 10.2 electron-rebuild 失败

**问题**: PowerShell 执行策略阻止 script 运行

```
File ... cannot be loaded because running scripts is disabled on this system.
```

**解决**:
```powershell
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned -Force
npx electron-rebuild
```

### 10.3 类型检查错误

**问题**: `vue-tsc --noEmit` 报错

**解决**:
1. 检查 `tsconfig.json` 路径别名配置是否正确
2. 检查工具函数 `src/utils/ipc-renderer.ts` 和 preload 中的类型定义一致性
3. 确保 `shared/types.ts` 中的 `ApiBridge` 与 preload 实现一致

### 10.4 数据库连接错误

**问题**: `Error: SQLITE_CANTOPEN: unable to open database file`

**解决**:
检查 `%APPDATA%\mlps-assessment-tool` 目录是否有写权限。首次运行时应用会自动创建目录结构。

### 10.5 端口冲突

**问题**: Vite 端口 5173 被占用

**解决**:
修改 `vite.config.ts` 中的 `server.port` 为其他端口：

```typescript
server: {
  port: 5174,  // 换一个端口
},
```

### 10.6 生产版本白屏

**问题**: 打包后应用启动白屏

**解决**:
1. 检查 `vite.config.ts` 中的 `base` 是否为 `'./'`（必须使用相对路径）
2. 检查 `electron/main/index.ts` 中加载路径是否正确
3. 在 Electron 主进程日志中查找页面加载错误

### 10.7 IPC 调用返回通道不存在

**问题**: `IPC channel ai:xxx not found`

**解决**:
检查 `electron/main/ipc.ts` 是否已注册新的 IPC 处理器，以及 `electron/preload/index.ts` 中是否暴露了对应的 API。

### 10.8 AI 截图上传后不显示

**问题**: 截图文件保存成功但 UI 无法加载

**解决**:
检查 `electron/ipc/screenshot.ipc.ts` 中的 `getBase64` 处理器是否正确读文件并返回 Base64 数据。
