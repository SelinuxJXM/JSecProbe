## 1. 项目概述

### 1.1 项目背景

等级保护现场测评系统是根据《GB/T 22239-2019 网络安全等级保护基本要求》等国家标准，为等级保护测评机构开发的专业现场测评工具。系统旨在规范测评流程、提高测评效率、保障测评质量，实现测评工作的数字化、标准化和智能化。

### 1.2 项目目标

- **标准化测评流程**: 依据国标要求，内置 GB/T 22239-2019 标准的 10 个安全域的测评项
- **智能化辅助**: 集成 AI 大模型能力，实现测评证据的智能分析和结论生成
- **全生命周期管理**: 覆盖项目创建、资产梳理、现场核查、问题管理、报告生成的完整流程
- **离线可用**: 基于 Electron 桌面架构，支持无网络环境下的现场测评工作
- **数据安全**: 本地加密存储，支持自动备份恢复

### 1.3 目标用户

| 用户角色 | 职责 | 核心需求 |
|---------|------|---------|
| 测评项目负责人 | 项目管理、进度监控 | 全局概览、数据统计 |
| 测评工程师 | 现场核查、证据采集 | 高效录入、AI辅助 |
| 技术专家 | 复杂问题分析 | 知识库检索、命令库 |
| 系统管理员 | 用户管理、系统配置 | 权限控制、数据安全 |

## 2. 系统架构设计

### 2.1 整体架构

系统采用 Electron 架构，分为 **主进程（Main Process）** 和 **渲染进程（Renderer Process）** 两大层级：

```
┌──────────────────────────────────────────────────────────────┐
│                    Electron 应用程序                          │
│  ┌─────────────────────────┐  ┌────────────────────────────┐ │
│  │    渲染进程 (Renderer)   │  │     主进程 (Main)          │ │
│  │  ┌───────────────────┐  │  │  ┌──────────────────────┐ │ │
│  │  │   Vue 3 + Vite    │  │  │  │   Node.js Runtime    │ │ │
│  │  │   Element Plus    │◄─┼──┼─►│   better-sqlite3     │ │ │
│  │  │   Pinia State     │  │  │  │   Drizzle ORM        │ │ │
│  │  │   Vue Router      │  │  │  │   File System        │ │ │
│  │  └───────────────────┘  │  │  └──────────────────────┘ │ │
│  └─────────────────────────┘  └────────────────────────────┘ │
│              ▲                            ▲                  │
│              │     Context Bridge          │                  │
│              │     (ipc-renderer)          │                  │
│              └────────────────────────────┘                  │
└──────────────────────────────────────────────────────────────┘
                              │
                    ┌─────────┴─────────┐
                    │                   │
              ┌─────▼─────┐      ┌─────▼─────┐
              │ SQLite DB │      │ File      │
              │ (better-  │      │ System    │
              │  sqlite3) │      │           │
              └───────────┘      └───────────┘
```

### 2.3 进程间通信（IPC）设计

系统采用 Electron 的 `ipcMain`/`ipcRenderer` 机制，通过 `contextBridge` 安全暴露 API：

```
Renderer (Vue)                    Main (Node.js)
    │                                  │
    │  ipcRenderer.invoke('channel')   │
    ├─────────────────────────────────►│
    │                                  │  业务处理
    │                                  │  ├── 数据库操作
    │                                  │  ├── 文件系统操作
    │                                  │  └── AI API 调用
    │  { success, data, error }        │
    │◄─────────────────────────────────┤
    │                                  │
```

**通信协议**:
- 请求/响应模式：`ipcMain.handle` / `ipcRenderer.invoke`
- 服务端推送：`mainWindow.webContents.send` + `ipcRenderer.on`
- 统一响应格式：`IpcResponse<T> = { success: boolean; data?: T; error?: { code, message } }`

## 5. 模块详细设计

### 5.1 认证授权模块

**功能职责**: 用户登录、密码修改、登录态管理

**技术实现**:
- 密码存储使用 `bcryptjs`（cost factor = 12）
- 登录态使用 LocalStorage 存储用户信息和 Token
- 路由守卫（Vue Router beforeEach）校验登录态

**IPC 通道**:
| 通道 | 方法 | 说明 |
|------|------|------|
| `auth:login` | POST | 用户名密码验证，返回用户信息 |
| `auth:logout` | POST | 清除登录态 |
| `auth:getCurrentUser` | GET | 获取当前用户 |

## 6. IPC 通信设计

### 6.1 通信规范

所有 IPC 通信遵循统一的请求-响应模式：

**请求格式**:
```typescript
ipcRenderer.invoke('module:action', ...args)
```

**响应格式**:
```typescript
interface IpcResponse<T> {
  success: boolean;
  data?: T;
  error?: {
    code: string;      // 错误码（大写蛇形）
    message: string;   // 可读错误信息
    details?: unknown; // 可选详情
  };
}
```

### 6.2 错误处理规范

主进程统一使用 `wrap` 高阶函数捕获异常：

```typescript
function wrap<T>(fn: () => T | Promise<T>): Promise<IpcResponse<T>> {
  return Promise.resolve()
    .then(fn)
    .then((data) => ({ success: true, data }))
    .catch((error) => ({
      success: false,
      error: { code: error.code || 'INTERNAL_ERROR', message: error.message }
    }));
}
```

## 7. 安全设计

### 7.1 架构安全

- **Context Isolation**: 启用 `contextIsolation: true`，渲染进程不直接暴露 Node.js API
- **Node Integration**: 禁用 `nodeIntegration: false`，防止渲染进程执行任意 Node 代码
- **Preload 脚本**: 通过 `contextBridge.exposeInMainWorld` 仅暴露必要 API
- **Web Security**: 启用 `webSecurity: true`，防止混合内容

### 7.2 路径安全

```typescript
async function validatePath(inputPath: string): Promise<string> {
  const resolved = path.resolve(inputPath);
  const allowedPaths = await getAllowedBasePaths();
  const isAllowed = allowedPaths.some(base => {
    const resolvedBase = path.resolve(base);
    return resolved === resolvedBase || resolved.startsWith(resolvedBase + path.sep);
  });
  if (!isAllowed) throw new Error('路径访问被拒绝');
  return resolved;
}
```

所有文件系统操作前必须通过路径验证，确保仅能访问应用数据目录。

### 7.3 密码安全

- 使用 bcrypt 算法（cost = 12）存储密码哈希
- 不使用明文存储或可逆加密
- 首次登录强制修改密码（`must_change_password` 标志）

### 7.4 数据安全

- 数据库文件存储在用户数据目录（`app.getPath('userData')`）
- 支持自动备份到独立备份目录
- 敏感配置（API Key）本地加密存储

### 8.2 自动备份策略

- **备份方式**: 直接文件复制（SQLite 文件级备份）
- **存储位置**: 用户数据目录下的 `backups/` 子目录
- **恢复机制**: 用户选择备份文件后覆盖当前数据库
