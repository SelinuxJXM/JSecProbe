## 1. 设计目标

### 1.1 系统定位

打造国内领先的等级保护现场测评桌面工具，实现测评工作的标准化、流程化和智能化。

### 1.2 核心目标

| 目标 | 说明 |
|------|------|
| 标准合规 | 完整内置等保 2.0 标准条款，自动适配适用性 |
| 高效便捷 | 多选粘贴、批量导入导出、AI 智能分析 |
| 数据驱动 | 历史项目分析、趋势预测、智能推荐 |
| 安全可靠 | 本地化部署、数据加密、操作审计 |

### 1.3 非功能目标

- **离线优先**: 不依赖云服务和网络
- **秒级响应**: 本地数据库、前端虚拟滚动
- **多实例支持**: 同时运行多个实例，数据完全隔离
- **便携部署**: 数据存储在程序同级目录，支持 U 盘便携使用

### 2.4 多实例支持设计

**设计理念**: 通过 PID 锁文件机制实现多实例共存，每个实例拥有独立数据目录。

```
启动流程:
1. 检查 locks 目录下的锁文件
2. 读取锁文件中的 PID，检测进程是否存活
3. 如果有活跃实例:
   - 第一个实例 → 默认目录 (JSecProbeData)
   - 第二个实例 → JSecProbeData_实例2
   - 第 N 个实例 → JSecProbeData_实例N
4. 创建当前实例的锁文件
5. 应用退出时自动清理锁文件
```

**数据存储路径（v1.2.5 更新）**:
| 场景 | 路径 |
|------|------|
| 开发模式 | `项目根目录\JSecProbeData\` |
| 生产模式 | `程序同级目录\JSecProbeData\` |
| 多实例 | 第二个实例自动使用 `JSecProbeData_实例2\` |

## 5. 模块详细设计

### 5.1 认证授权模块

**功能职责**: 用户登录、密码修改、登录态管理

**技术实现**:
- 密码存储使用 `bcryptjs`（cost factor = 12）
- 登录态使用 LocalStorage 存储用户信息和 Token
- 路由守卫（Vue Router beforeEach）校验登录态

**IPC 通道**:
| 通道 | 方法 | 说明 |
|------|------|------|
| `auth:login` | POST | 用户名密码验证，返回用户信息 |
| `auth:logout` | POST | 清除登录态 |
| `auth:getCurrentUser` | GET | 获取当前用户 |
| `auth:changePassword` | POST | 修改密码（验证旧密码） |

### 5.2 项目管理模块

**功能职责**: 项目创建、编辑、列表查询、导入导出

**核心流程**:
1. 填写项目基本信息（名称、系统名、保护等级、扩展要求等）
2. 系统自动加载对应等级和扩展类型的测评项（从数据库筛选 `minLevel <= projectLevel` 且 `extensionType IN ('general', projectExtensions)` 的测评项）
3. 项目生命周期：草稿 → 进行中 → 已完成 → 已归档

**IPC 通道**:
| 通道 | 说明 |
|------|------|
| `project:list` | 分页列表查询 |
| `project:get` | 获取详情 |
| `project:create` | 创建项目 |
| `project:update` | 更新项目 |
| `project:remove` | 删除项目 |
| `project:export` | 导出项目数据 |
| `project:exportAll` | 导出全部项目 |

### 5.3 系统构成模块（资产管理）

**功能职责**: 资产信息录入、分类管理、Excel 导入导出

**资产类别体系**:
- `server_storage` - 服务器/存储设备
- `dbms` - 数据库管理系统
- `network_device` - 网络设备
- `security_device` - 安全设备
- `business_app` - 业务应用系统
- `terminal` - 终端设备（业务终端/运维终端）
- `management_platform` - 管理平台
- `machine_room` - 机房设施
- `data_resource` - 数据资源
- `network_boundary` - 网络边界

**Excel 导入设计（v1.2.5 更新）**:
- 支持多 Sheet 导入，每个 Sheet 名称对应一个资产类别
- 精确列匹配（Exact Column Matching）：根据导出的表头名称精确匹配字段
- 支持模糊匹配回退（精确匹配失败时按关键词匹配）
- Sheet 名称映射：
  - '管理机房' → machine_room
  - '服务器/存储设备' / '服务器存储设备' / '服务器-存储设备' → server_storage
  - '业务终端/运维终端' / '业务终端运维终端' / '业务终端-运维终端' / '运维终端' / '业务终端' / '终端' → terminal
  - 更多映射...

**字段映射**:

| 操作系统及版本/系统及版本/软件及版本/所在设备名称 | os |
| 品牌及型号/版本/类型/版本 | version |
| IP地址 | ip |
| 数量 | quantity |
| 备注/描述 | description |
| 重要程度 | importance |
| 虚拟设备 | isVirtual |
| 测评对象 | isAssessmentTarget |
| 主要功能/设备用途 | deviceUsage |

## 6. IPC 通信设计

### 6.1 通信规范

所有 IPC 通信遵循统一的请求-响应模式：

**请求格式**:
```typescript
ipcRenderer.invoke('module:action', ...args)
```

**响应格式**:
```typescript
interface IpcResponse<T> {
  success: boolean;
  data?: T;
  error?: {
    code: string;      // 错误码（大写蛇形）
    message: string;   // 可读错误信息
    details?: unknown; // 可选详情
  };
}
```

### 6.2 错误处理规范

主进程统一使用 `wrap` 高阶函数捕获异常：

```typescript
function wrap<T>(fn: () => T | Promise<T>): Promise<IpcResponse<T>> {
  return Promise.resolve()
    .then(fn)
    .then((data) => ({ success: true, data }))
    .catch((error) => ({
      success: false,
      error: { code: error.code || 'INTERNAL_ERROR', message: error.message }
    }));
}
```

## 7. 安全设计

### 7.1 架构安全

- **Context Isolation**: 启用 `contextIsolation: true`，渲染进程不直接暴露 Node.js API
- **Node Integration**: 禁用 `nodeIntegration: false`，防止渲染进程执行任意 Node 代码
- **Preload 脚本**: 通过 `contextBridge.exposeInMainWorld` 仅暴露必要 API
- **Web Security**: 启用 `webSecurity: true`，防止混合内容

### 7.2 路径安全

```typescript
async function validatePath(inputPath: string): Promise<string> {
  const resolved = path.resolve(inputPath);
  const allowedPaths = await getAllowedBasePaths();
  const isAllowed = allowedPaths.some(base => {
    const resolvedBase = path.resolve(base);
    return resolved === resolvedBase || resolved.startsWith(resolvedBase + path.sep);
  });
  if (!isAllowed) throw new Error('路径访问被拒绝');
  return resolved;
}
```

所有文件系统操作前必须通过路径验证，确保仅能访问应用数据目录。

### 7.3 密码安全

- 使用 bcrypt 算法（cost = 12）存储密码哈希
- 不使用明文存储或可逆加密
- 首次登录强制修改密码（`must_change_password` 标志）

### 7.4 数据安全

- 数据库文件存储在程序同级目录（`JSecProbeData\data\mlps.db`）
- 支持自动备份到独立备份目录
- 敏感配置（API Key）本地存储
- 多实例启动时，每个实例拥有独立数据目录，互不干扰

### 8.2 数据存储路径配置

```json

  "autoBackupEnabled": 1,

  "ocrProvider": "local"
```

**路径优先级**:
1. 如果 `app-config.json` 中存在 `dataPath`，使用该路径

### 8.3 自动备份策略

- **触发时机**: 应用启动时检查 + 每 6 小时定时检查
- **备份条件**: 距上次备份超过 `autoBackupDays`（默认 7 天）
- **备份方式**: 直接文件复制（SQLite 文件级备份）
- **存储位置**: 数据目录下的 `backups/` 子目录
- **恢复机制**: 用户选择备份文件后覆盖当前数据库
