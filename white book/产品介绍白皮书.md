## 1. 产品概述

### 1.1 产品定位

等级保护现场测评系统（以下简称"本系统"）是面向等级保护测评机构的专业桌面工具软件。本系统严格依据《GB/T 22239-2019 网络安全等级保护基本要求》等国家标准设计，旨在为测评工程师提供覆盖测评全生命周期的数字化工作台，实现测评工作的标准化、流程化和智能化。

### 1.2 产品愿景

成为中国等级保护测评行业的首选数字化工具，推动测评工作从"经验驱动"向"数据驱动"转型，从"人工操作为主"向"AI 智能辅助"升级。

### 1.3 核心价值

| 价值维度 | 传统方式 | 本系统 |
|---------|---------|--------|
| 测评效率 | 手工记录、纸质文档，单个项目耗时 2-4 周 | 数字化录入、AI 辅助分析，效率提升 50%+ |
| 标准执行 | 依赖个人经验，标准理解不一致 | 内置国标测评项，自动匹配适用条款 |
| 证据管理 | 分散存储、易丢失 | 集中管理、自动归档、一键导出 |
| 质量管控 | 难以追溯、复核困难 | 全程留痕、进度可视、结论可追溯 |
| 知识积累 | 个人经验、难以共享 | 知识库沉淀、团队协同 |

## 2. 市场分析

### 2.1 政策背景

根据《中华人民共和国网络安全法》和《网络安全等级保护条例》，二级以上信息系统必须定期开展等级保护测评。随着等保 2.0 标准的全面实施，测评市场需求持续增长：

- **测评机构数量**: 全国等级保护测评机构超过 500 家
- **年度测评量**: 全国年度测评项目超过 10 万个
- **市场规模**: 等级保护测评服务市场规模超过 50 亿元/年

### 2.2 行业痛点

| 痛点 | 描述 | 影响 |
|------|------|------|
| 标准执行不统一 | 不同测评师对标准理解存在差异 | 测评结论一致性差 |
| 现场工作低效 | 手工记录、重复录入 | 项目周期长、成本高 |
| 证据管理混乱 | 截图、文档分散存储 | 复核困难、易丢失 |
| 知识传承困难 | 经验依赖个人 | 新人上手慢、团队协作难 |
| 报告质量参差 | 手工编写、格式不统一 | 报告质量难以保证 |

### 2.3 目标用户

**主要用户**: 等级保护测评机构及其测评工程师

**用户画像**:
- 测评机构技术负责人：关注项目管理、质量管控
- 高级测评工程师：关注效率提升、AI 辅助
- 初级测评工程师：关注标准引导、知识参考
- 机构管理员：关注用户管理、数据安全

## 3. 产品架构

### 3.1 技术架构

本系统采用 Electron 桌面应用架构，结合 Vue 3 前端框架和 SQLite 本地数据库，实现跨平台、高性能、易部署的桌面应用。

```
┌─────────────────────────────────────────────────────────┐
│                    等级保护现场测评系统                     │
│  ┌─────────────────────────────────────────────────────┐│
│  │              前端展示层 (Vue 3 + Element Plus)       ││
│  │   工作台 │ 项目管理 │ 现场核查 │ 问题管理 │ 知识库   ││
│  └────────────────────────┬────────────────────────────┘│
│                           │ IPC (Context Bridge)        │
│  ┌────────────────────────┴────────────────────────────┐│
│  │              业务逻辑层 (Electron Main Process)      ││
│  │   认证 │ 项目 │ 资产 │ 测评 │ 问题 │ AI │ 报告     ││
│  └────────────────────────┬────────────────────────────┘│
│                           │                             │
│  ┌────────────────────────┴────────────────────────────┐│
│  │              数据存储层 (SQLite + File System)       ││
│  │   结构化数据 (SQLite) │ 文件数据 (截图/文档/报告)    ││
│  └─────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────┘
```

### 3.2 部署架构

本系统采用**纯本地化部署**模式，无需服务器支持：

- **数据存储**: 本地 SQLite 数据库 + 文件系统
- **AI 调用**: 通过配置对接外部 AI API（OpenAI 兼容格式）
- **更新机制**: 支持在线更新和离线安装包分发
- **备份恢复**: 本地自动备份 + 手动导出

## 4. 核心功能

### 4.1 工作台（Dashboard）

工作台为用户提供项目全局概览和快速操作入口：

- **项目统计**: 项目总数、进行中项目、已完成项目
- **进度概览**: 各项目测评进度可视化
- **待办事项**: 待处理问题、待完成测评项
- **快捷操作**: 新建项目、继续上次工作

### 4.2 项目管理

全生命周期项目管理功能：

- **项目创建**: 选择标准体系、保护等级、扩展要求
- **项目信息**: 被测单位、系统名称、测评时间、负责人
- **项目状态**: 草稿 → 进行中 → 已完成 → 已归档
- **项目导入导出**: 支持项目数据打包迁移

### 4.3 系统构成（资产管理）

资产信息管理与分类体系：

- **10 大资产类别**: 服务器/存储、数据库、网络设备、安全设备、业务应用、终端、管理平台、机房、数据资源、网络边界
- **资产属性**: IP 地址、操作系统、版本、责任人、重要性等级
- **Excel 导入**: 支持批量导入资产清单
- **资产统计**: 按类别、重要性分布统计

### 4.4 现场核查（核心功能）

现场核查是系统的核心模块，支持两种测评模式：

#### 4.4.1 层面测评

针对安全域整体进行测评，不关联具体资产：

1. 选择安全域（如"安全计算环境"）
2. 系统自动加载该域下所有适用测评项
3. 逐项执行核查：选择方法（访谈/检查/测试）、录入结果、上传证据
4. 判定符合情况：符合 / 部分符合 / 不符合 / 不适用

#### 4.4.2 资产测评

针对具体资产执行测评：

1. 选择目标资产
2. 系统根据资产类别自动匹配适用测评项
3. 执行核查并关联到资产
4. 支持跨资产批量测评

#### 4.4.3 证据管理

- **截图上传**: 支持文件选择、粘贴板粘贴、Base64 直接保存
- **文件附件**: 支持 PDF、Word、Excel 等格式
- **命令输出**: 记录核查命令及输出结果
- **证据关联**: 自动关联到对应测评项和资产

#### 4.4.4 进度与统计

- **实时进度**: 自动计算并显示测评完成百分比
- **符合率分析**: 按安全域统计符合率
- **风险分布**: 高风险项识别与统计

### 4.5 AI 智能辅助

系统集成 AI 大模型能力，提供智能分析辅助：

#### 4.5.1 智能分析

- **输入**: 测评项要求 + 核查证据（文本/截图）
- **输出**: 关键证据点提取、符合性判定、测评结论
- **特点**: 
  - 严格基于证据分析，避免编造
  - 结论以"经核查"/"经访谈"开头，符合测评规范
  - 支持多模态输入（文本 + 图片）

#### 4.5.2 批量分析

- **输入**: 多张截图 + 多个测评项
- **智能匹配**: AI 自动将截图与对应测评项匹配
- **进度可视化**: 实时显示分析进度（编码→发送→接收→解析→完成）
- **结果导出**: 一键应用分析结果到测评记录

#### 4.5.3 AI 配置

- **兼容 OpenAI API**: 支持任何兼容 OpenAI 格式的 API
- **可配置参数**: API Base、API Key、模型名称、温度参数
- **连接测试**: 一键测试 API 连通性

### 4.6 问题管理

问题全生命周期跟踪管理：

- **问题生成**: 从测评记录自动生成不符合项
- **问题属性**: 风险等级（高/中/低）、安全域、控制点、关联资产
- **整改跟踪**: 待处理 → 整改中 → 已整改 → 已关闭
- **统计分析**: 按风险等级、安全域、状态分布统计
- **Excel 导出**: 导出问题清单

### 4.7 报告生成

一键生成测评报告：

- **报告内容**: 项目信息、测评统计、问题清单
- **输出格式**: PDF
- **自动归档**: 报告自动保存到项目目录

### 4.8 知识库

内置专业知识库，辅助现场测评：

- **知识分类**: 测评方法、操作系统、网络设备、安全设备、数据库、中间件、管理规范
- **文档管理**: 支持上传 PDF、Word、Excel 等格式文档
- **命令库**: 收录常用核查命令，支持按操作系统/厂商/设备类型检索
- **收藏功能**: 常用命令一键收藏

### 4.9 系统设置

系统管理与配置：

- **用户管理**: 用户增删改查、角色分配
- **AI 配置**: API 参数配置与连接测试
- **数据备份**: 自动备份（可配置周期）+ 手动备份
- **数据恢复**: 从备份文件恢复
- **日志管理**: 操作日志查看

## 5. 技术亮点

### 5.1 离线优先架构

- **本地数据库**: SQLite 零配置，无需数据库服务
- **文件系统存储**: 截图、证据、报告本地化管理
- **无网络依赖**: 核心功能完全离线可用
- **AI 可选**: AI 辅助为可选功能，不依赖也可完成全部测评工作

### 5.2 智能测评辅助

- **自动匹配**: 根据项目等级和扩展要求自动筛选适用测评项
- **资产映射**: 根据资产类别自动匹配相关安全域和测评项
- **AI 分析**: 大模型辅助证据分析和结论生成
- **批量处理**: 支持截图批量分析和结果自动匹配

### 5.3 数据安全保障

- **本地加密**: 密码 bcrypt 哈希存储
- **路径安全**: 文件系统操作限制在应用数据目录
- **自动备份**: 定时自动备份，防止数据丢失
- **操作审计**: 完整操作日志记录

### 5.4 高效交互设计

- **键盘友好**: 支持快捷键操作，提高录入效率
- **批量操作**: 支持批量导入、批量导出、批量处理
- **智能筛选**: 多维度筛选和搜索
- **可视化**: 进度图表、统计图表直观展示

### 5.5 标准合规

- **国标内置**: GB/T 22239-2019 标准测评项完整内置
- **扩展要求**: 支持云计算、移动互联、物联网、工业控制、大数据等扩展要求
- **等级适配**: 自动适配二级、三级、四级测评项差异

## 7. 应用场景

### 7.1 等级保护测评

**场景描述**: 测评机构执行等保测评项目

**使用流程**:
1. 创建项目 → 录入被测系统信息
2. 资产梳理 → 导入或录入资产清单
3. 现场核查 → 逐项执行测评、采集证据
4. AI 辅助 → 上传截图，AI 自动分析
5. 问题汇总 → 自动生成问题清单
6. 报告输出 → 一键生成测评报告

### 7.2 安全自查

**场景描述**: 系统运营单位开展等保自查

**使用流程**:
1. 创建自查项目
2. 录入自有资产
3. 按测评项逐项自查
4. 参考知识库中的配置方法
5. 生成自查报告和问题清单

### 7.3 测评机构内部管理

**场景描述**: 测评机构管理多个项目和团队

**使用流程**:
1. 管理员创建用户账号
2. 项目负责人创建项目并分配成员
3. 测评工程师执行各自负责的测评任务
4. 管理员查看全局进度和质量

### 7.4 知识积累与培训

**场景描述**: 机构内部知识沉淀和新人培训

**使用流程**:
1. 将优秀测评案例录入知识库
2. 整理常用命令和配置方法
3. 新人通过知识库学习测评方法
4. 统一测评标准和结论格式
